# 智慧科技园区无人巡检与安防调度系统

基于 Unity 3D 实现的多机器人巡检与安防调度课程项目。系统将科技园区道路抽象为带权图，通过优先队列管理巡检与紧急任务，使用 D* Lite 和 A* 完成路径规划及动态重规划，并提供异常事件、低电量充电、拥堵模拟、空间查询、监控面板和压力测试等功能。

## 项目功能

- 支持 4 台巡检机器人独立维护 ID、电量、传感器状态、当前任务和待执行任务队列。
- 支持定时巡检、定点巡查、异常响应、紧急事件和充电任务。
- 高优先级任务可插队，并在允许打断时暂停当前任务；被暂停任务会重新进入任务队列。
- 路网支持距离、方向、拥堵系数、临时封闭、夜间通行和主干道标记。
- 优先使用 D* Lite 规划路径，规划失败时自动回退到 A*。
- 道路拥堵或封闭发生变化后，通知执行中机器人重新规划路线。
- 使用轨迹栈记录机器人经过的安全位置，规划失败时可回退。
- 使用网格空间索引查询附近充电桩、异常点和机器人。
- 低电量机器人自动前往附近充电桩。
- 监控面板实时显示机器人电量、任务、队列、传感器状态和规划耗时。
- 摄像机默认跟随 `R-01`，可通过顶部按钮切换跟随 `R-01` 至 `R-04`。
- 提供 225 节点、50 个异常事件、100 个任务的压力测试，以及道路封闭后的 10 秒重规划测试。

## 数据结构与算法

| 模块 | 数据结构或算法 | 作用 |
| --- | --- | --- |
| 园区路网 | 带权图、邻接表 | 表示道路节点和道路连接关系，支持动态边权与封路 |
| 路径规划 | D* Lite、A* | 结合距离、拥堵、能耗和夜间限制计算路径 |
| A* 开放集 | 二叉堆 | 快速获取当前代价最小的候选节点 |
| 机器人任务 | 优先队列（二叉堆） | 按“优先级优先、时间戳次优先”的规则调度任务 |
| 行驶记录 | 栈 | 保存机器人经过的位置，支持原路回退到安全点 |
| 当前路径 | 队列 | 按顺序保存和消费机器人下一步要到达的路径点 |
| 异常事件 | 全局队列 | 按事件到达顺序交给调度器处理 |
| 机器人索引 | 哈希表 | 通过机器人 ID 快速查找对应对象 |
| 区域查询 | 网格空间索引 | 查询附近机器人、充电桩及异常目标 |

道路基础代价由物理距离与拥堵共同决定：

```text
道路代价 = 距离 × (1 + 2 × 拥堵系数)
```

路径搜索还会根据机器人的剩余电量和能耗权重增加代价。临时封闭道路不可通行；夜间模式下，`NightAllowed = false` 的道路也不可通行。

## 系统工作流程

```text
定时任务 / 异常事件 / 低电量事件
                 ↓
          DispatchManager
                 ↓
      选择综合代价较低的机器人
                 ↓
      PriorityTaskQueue 任务入队
                 ↓
             RobotAgent
                 ↓
        D* Lite 规划，A* 兜底
                 ↓
         路径点队列驱动机器人移动
                 ↓
   路况变化时重新规划，失败时使用轨迹栈回退
```

## 快速运行

1. 使用 Unity Hub 添加本项目根目录。
2. 使用 Unity `2022.3.62f1c1` 或兼容的 Unity 2022.3 LTS 版本打开项目。
3. 等待 Unity 完成资源导入和脚本编译。
4. 打开场景 `Assets/Scenes/SampleScene.unity`。
5. 确认 Console 中没有编译错误，然后点击 Play。

项目场景已经配置好 4 台机器人、路网、调度器、异常生成器、监控面板、摄像机跟随和验收测试对象，正常情况下无需额外绑定即可运行。

## 运行操作

| 操作 | 功能 |
| --- | --- |
| 点击顶部 `R-01` 至 `R-04` | 切换摄像机跟随目标，当前机器人按钮为灰色且不可重复点击 |
| `F2` | 显示或隐藏监控面板 |
| `F3` | 手动生成一条异常事件 |
| `F4` | 显示或隐藏道路拥堵热力图 |
| `F8` | 显示或隐藏机器人切换按钮 |

监控面板还提供以下测试按钮：

- 场景图 A* 随机规划 200 次。
- 完整验收测试：压力测试和容错测试。
- 压力测试：构造 `15 × 15` 的 225 节点路网，执行随机寻路，并注入 50 个异常事件及 100 个任务。
- 容错测试：随机封闭主干道，检查执行中机器人能否在 10 秒内恢复有效路径。

## 摄像机设置

`Main Camera` 上挂载了 `RobotCameraFollower`。运行后，摄像机会成为当前机器人对象的子物体，并使用以下相对参数：

- `DefaultRobotIndex`：默认跟随机器人下标，`0` 对应 `R-01`。
- `LocalPosition`：摄像机相对机器人的位置，当前为 `(0, 2.2, -3.5)`。
- `LocalEulerAngles`：摄像机相对机器人的旋转，当前为 `(12, 0, 0)`。
- `ButtonStart`、`ButtonSize`、`ButtonGap`：切换按钮的位置、大小和间距。

需要调整视角时，在 Inspector 中修改 `LocalPosition` 和 `LocalEulerAngles` 即可。切换机器人后仍会使用同一组相对坐标。

## 路网与动态环境设置

`GraphManager` 上的 `WeightedGraph` 维护节点和边：

- 节点的 `Point` 绑定场景中的 RoadNode，节点 ID 按列表顺序自动编号。
- `FromId`、`ToId` 指定道路两端节点。
- `TwoWay` 控制道路是否双向。
- `Congestion` 表示 `0~1` 的实时拥堵程度。
- `Closed` 表示道路是否临时封闭。
- `NightAllowed` 表示道路是否允许夜间通行。
- `IsMainRoad` 用于主干道封闭与容错测试。

修改运行时道路状态后，需要调用 `WeightedGraph.Rebuild()` 更新邻接表，再由 `DispatchManager.NotifyRoadChanged()` 通知机器人重新规划。

## 路径显示

机器人路线由 `RobotPathVisualizer` 负责绘制。当前项目按演示需求将 `ShowPath` 默认设为 `false`，因此运行时不会显示路线。需要展示普通路径和红色紧急路径时，可在机器人 Inspector 中重新启用 `ShowPath`。

## 核心脚本

| 脚本 | 职责 |
| --- | --- |
| `WeightedGraph.cs` | 构建园区带权图、邻接表及路网版本，维护道路状态 |
| `AStarPathfinder.cs` | 多权重 A* 寻路和规划耗时统计 |
| `DStarLitePathfinder.cs` | D* Lite 增量路径规划 |
| `PathfindingHeap.cs` | 寻路开放集使用的二叉堆 |
| `PathSearchOptions.cs` | 夜间、能耗和电量等寻路参数 |
| `PatrolTask.cs` | 任务类型、优先级、时间戳和目标信息 |
| `PriorityTaskQueue.cs` | 基于堆的机器人任务优先队列 |
| `RobotAgent.cs` | 机器人状态、任务执行、移动、动画、重规划与回退 |
| `DispatchManager.cs` | 多机器人任务生成、分配、异常调度及充电调度 |
| `AnomalyEvent.cs` | 火灾、烟雾、异常声音、交通违章等异常事件模型 |
| `GlobalAnomalyQueue.cs` | 全局异常事件队列 |
| `AnomalyEventSpawner.cs` | 模拟实时异常上报 |
| `RoadEventSimulator.cs` | 动态修改拥堵和道路封闭状态 |
| `SpatialGridIndex.cs` | 网格空间索引与附近资源查询 |
| `ChargingStation.cs` | 充电桩标记及充电逻辑 |
| `RobotCameraFollower.cs` | 摄像机跟随与 4 台机器人切换按钮 |
| `MonitorDashboard.cs` | 运行状态、任务、电量、报警和测试界面 |
| `CongestionHeatmapVisualizer.cs` | 道路拥堵热力图 |
| `HomeworkVerificationRunner.cs` | 压力测试与道路封闭容错测试 |
| `RobotPathVisualizer.cs` | 可选的路径可视化 |

## 资源与 AI 辅助说明

- 园区及环境资源主要参考或来源于 Sketchfab，具体使用时遵循对应资源页面的授权要求。

## 当前说明

- 当前场景使用 4 台机器人，机器人数量可通过 `DispatchManager.Robots` 列表调整。
- 路径可视化默认关闭，拥堵热力图可在运行时按 `F4` 切换。
- 当前动态避障以路网拥堵、道路封闭和重新规划为主，尚未实现机器人之间基于时间窗的精细避碰。
